Operations

config

Set the user name for the current repository to UserName:

git config user.name "UserName"
git config --global user.name "UserName"
git config --global user.email "RemoteEmail"
git config -l # list all

init

Initialize Git on the current folder.

git init

clone

git clone https://github.com/aaabbbccc/political_platform.git

status

git status

Possible result … untracked file

On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	case-2.sh

nothing added to commit but untracked files present (use "git add" to track)

Check the status, but see the changes in a more compact way.

git status --short

Short status flags

  • ?? - Untracked files
  • A - Files added to stage
  • M - Modified files
  • D - Deleted files

add

AddÂ clear.htmlÂ to the Staging Enviornment:

git add clear.html

Stage all new, modified, and deleted files.

git add .
git add --all
git add -A #shorthand

commit

  • Git considers eachÂ commitÂ change point or “save point”. It is a point in the project you can go back to if you find a bug, or want to make a change.
  • ==When we commit, we should always include a message.==
  • By adding clear messages to eachÂ commit, it is easy for yourself (and others) to see what has changed and when.

Commit the changes to the current repository with the message “First release!”

git commit -m "First release!"

Commit the updated files directly, skipping the staging environment:

git commit -a -m "First release! (staging skipped)"

branch

Create a new branch called hello-world-images.

git branch hello-world-images

List the existing branches.

git branch

* beside master specifies that we are currently on that branch.

Delete an old branch.

git branch --delete old-branch.
git branch -d old-branch.

checkout

Check out aÂ branch = Switch from another branch to that branch.

Check out the branch

git checkout hello-world-images

Out

Switched to branch 'br1'

Create, and move to a new branch with the name hello-you

git checkout -b hello-you

merge

git merge hello-world-images

remote

Add an remote repository.

git remote add <remote-name> <url>

List remote repositories.

git remote -v

Add an remote repository and push

git remote set-url --add --push origin <url>

push

git push <remote-name> <branch-name>
git push origin main

pull

Git Pull is a combination of fetch and merge.

Fetch remote branches and merge them with local repos

git pull --all

diff

blame

Show what revision and author last modified each line of a file called foo

git blame foo

help

See all the available options for the specific command (e.g. commit.)

git commit -help

List all possible commands. (This will display a very long list of commands!)

git help --all
